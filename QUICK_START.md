# 快速开始：真实数据源配置

本指南帮助您快速配置真实数据源，开始抓取公众号文章。

## 快速配置步骤

### 使用搜狗微信搜索（5分钟）

#### 1. 打开小程序
- 启动"公众号内容聚合助手"小程序

- 查看抓取结果

- 文章按关键词分组展示
- 点击文章可复制链接
- 在浏览器中打开链接查看完整内容

### 使用第三方API（30分钟）

#### 1. 选择服务商

推荐以下服务商（任选其一）：

**新榜（NewRank）**
- 官网：https://www.newrank.cn/
- 特点：数据全面、更新及时
- 价格：联系客服咨询

**清博大数据**
- 官网：https://www.gsdata.cn/
- 特点：数据分析功能强大
- 价格：联系客服咨询

**微小宝**
- 官网：https://www.wxb.com/
- 特点：专注公众号数据
- 价格：联系客服咨询

#### 2. 注册并购买服务

1. 访问服务商官网
2. 注册账号
3. 购买API服务套餐
4. 获取API Key

#### 3. 配置API密钥

**方式一：通过Supabase Dashboard（推荐）**

1. 登录Supabase Dashboard
2. 进入项目设置 → Edge Functions → Secrets
3. 添加环境变量：
   - Key: `THIRD_PARTY_API_KEY`
   - Value: 你的API密钥
4. 保存

**方式二：通过小程序配置**

1. 打开小程序
2. 进入"配置"页面
3. 点击"数据源配置"
4. 选择"第三方API"
5. 输入API密钥
6. 点击"保存配置"

#### 4. 修改Edge Function

编辑 `supabase/functions/fetch_articles_real/index.ts`：

```typescript
async function fetchFromThirdPartyAPI(
  keyword: string,
  accountName: string,
  apiKey?: string
): Promise<any[]> {
  // 根据你选择的服务商修改以下代码
  
  // 示例：新榜API
  const response = await fetch('https://api.newrank.cn/v1/articles/search', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${apiKey}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      keyword: keyword,
      account: accountName,
      limit: 5
    })
  })
  
  const data = await response.json()
  
  // 转换为统一格式
  return data.articles.map(article => ({
    title: article.title,
    summary: article.abstract,
    link: article.url,
    published_at: article.publish_time
  }))
}
```

#### 5. 部署Edge Function

```bash
cd /workspace/app-apbcv7yhhuyp
supabase functions deploy fetch_articles_real
```

#### 6. 测试抓取

1. 打开小程序
2. 进入"简报"页面

## 常见问题排查

### 问题1：搜狗微信搜索抓取失败

**错误提示**：
```
抓取失败，请稍后重试
未找到新文章
成功: 0 | 失败: 3
```

**可能原因**：
1. IP被封禁
2. 网络问题
3. HTML结构变化

**解决方案**：
1. 等待30分钟后重试
2. 检查网络连接
3. 切换到第三方API
4. 使用模拟数据测试功能

---

### 问题2：第三方API返回错误

**错误提示**：
```
插入文章失败: API authentication failed
```

**可能原因**：
1. API密钥错误
2. 账户余额不足
3. API限流

**解决方案**：
1. 检查API密钥是否正确
2. 登录服务商后台查看余额
3. 降低请求频率
4. 联系服务商技术支持

---

### 问题3：抓取速度太慢

**现象**：
- 搜狗微信搜索需要30秒以上
- 页面长时间显示"抓取中..."

**原因**：
- 每个关键词和公众号组合都需要单独请求
- 添加了1秒延迟避免被封禁
- 网络延迟

**优化方案**：
1. 减少关键词和公众号数量
2. 使用第三方API（速度更快）
3. 实现缓存机制（避免重复抓取）

---

### 问题4：文章重复

**现象**：
- 同一篇文章出现多次
- 刷新后文章数量不断增加

**原因**：
- 去重逻辑失效
- 文章标题不完全一致

**解决方案**：
1. 清空历史文章（删除旧数据）
2. 优化去重逻辑（使用文章链接去重）
3. 定期清理7天前的文章

## 性能优化建议

### 1. 合理配置关键词和公众号

**推荐配置**：
- 关键词：3-5个
- 公众号：5-10个
- 总请求数：15-50次

**计算公式**：
```
总请求数 = 关键词数量 × 公众号数量
抓取时间 = 总请求数 × (请求时间 + 延迟时间)
```

**示例**：
- 3个关键词 × 5个公众号 = 15次请求
- 15次请求 × (1秒 + 1秒) = 30秒

### 2. 使用缓存机制

**实现思路**：
1. 记录上次抓取时间
2. 如果距离上次抓取不足1小时，直接返回缓存
3. 超过1小时才重新抓取

### 3. 实现增量更新

**实现思路**：
1. 记录每篇文章的发布时间
2. 只抓取比最新文章更新的内容
3. 避免重复抓取旧文章

### 4. 使用并发请求

**实现思路**：
1. 使用 `Promise.all()` 并发请求
2. 设置最大并发数（避免被封禁）
3. 实现请求队列
